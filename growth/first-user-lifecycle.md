# BriefLink first-user lifecycle

## Purpose
Lifecycle messaging for the first cohort of BriefLink users.

## Messaging principles
- Keep copy concise and credible
- Emphasize clarity, alignment, and easier follow-through on client work
- Avoid unsupported feature or integration claims
- Use SMS/in-app only where a short prompt adds value

---

## 1) Waitlist confirmation
**Email subject:** You’re on the BriefLink early list

**Email**
Hi `{first_name}`,

You’re in.

BriefLink is built for agencies and consultancies that want a clearer way to handle client input and turn it into something useful for the team.

We’ll send updates and access details as spots open.

— `{sender_name}`

**SMS / in-app**
You’re on the BriefLink early list. We’ll share updates and access details as spots open.

## 2) Welcome
**Email subject:** Welcome to BriefLink

**Email**
Hi `{first_name}`,

The best place to start is with one real piece of client work your team is already handling. That makes it easier to judge whether BriefLink fits your workflow.

Welcome to BriefLink. Start with one real piece of client work so the value is easier to judge.

## 3) Activation nudge
**Trigger:** No meaningful action after 2 days  
**Email subject:** A simple way to try BriefLink

**Email**

A quick nudge in case BriefLink is still sitting unopened.

The easiest way to evaluate it is with something real your team already needs to clarify, organize, or share.

**SMS / in-app**
Try BriefLink with one real piece of current client work. That’s the fastest way to assess fit.

## 4) Value reminder

BriefLink tends to be most useful when a team wants clearer shared understanding around active client work.

If you already have something this week that feels messy or unclear, that’s usually the best time to try it again.
