# BriefLink growth ops notes

## Package intent
These assets are written for the current launch cycle using the approved in-chat canon only:
- BriefLink MVP framing
- clarity-first, product-credible brand voice
- agency/consultancy buyer focus
- approved launch structure

Use this package as the recommended default until later artifact-specific tightening is needed.

## Segmentation assumptions
Primary audience:
- agencies
- consultancies
- studios and service firms doing custom client work

Priority roles:
- founder
- managing partner
- principal
- operations lead
- client services lead
- delivery lead

## Send guidance
- Keep founder outreach plain-text friendly
- Use 1–2 personalization tokens max per outbound message
- Keep lifecycle messages short and operational
- Favor email by default; use SMS/in-app only for short prompts
- Do not broaden messaging to general SMB productivity

## Manual-fill placeholders
Replace before send:
- `{first_name}`
- `{firm_name}`
- `{specialization}`
- `{shared_connection}`
- `{sender_name}`

Optional if available:
- booking link
- waitlist link
- reply-to address

## Claims to avoid
Do not add claims about:
- integrations
- advanced automation
- analytics
- broad workflow coverage beyond clarity for client work
- guaranteed performance outcomes

## Recommended usage order
1. Use `founder-outreach.md` for initial outreach
2. Use `first-user-lifecycle.md` once a prospect joins or gets access
3. Use `funnel-map.md` for stage-to-channel reference
4. Use `launch-bundle.md` as the package overview
