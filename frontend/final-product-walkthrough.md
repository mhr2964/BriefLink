# Step-by-Step Flow
1. Create brief — user pastes source text and submits it.
2. Share — user reviews the generated brief and hands it off.
3. Recipient consumption — recipient opens and reads the shared brief.

# What the Product Demonstrates
- Create brief: raw text becomes a concise, structured brief; input/loading/success/error states stay clear.
- Share: the generated brief is ready to hand off without rewriting; the same artifact continues downstream.
- Recipient consumption: the recipient sees the same distilled brief in a lightweight readable format.

# Screens, States, and Assets
- Create screen: source-text input and primary submit action.
- Generation states: loading, success, error.
- Brief view: title, summary, highlights, next steps.
- Share state/action: handoff anchored to the generated brief.
- Recipient view: read-only version of the same brief structure.
- Core asset: one stable brief format reused from creation through recipient consumption.
